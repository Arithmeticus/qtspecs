1 Introduction

This document describes the namespace http://www.w3.org/2007/xpath-full-text defined by the [XQuery and XPath Full-Text 1.0] specification (Recommendation of 13 March 2011). This namespace is typically identified by the namespace prefix xqxft:, which is not a predefined prefix. For updated information, please refer to the latest version of the [XQuery and XPath Full-Text 1.0] specification.

This document contains a directory of links to related resources, using RDDL (as defined in [Resource Directory Description Language (RDDL)]).

It is GRDDL-enabled (as defined in [Gleaning Resource Descriptions from Dialects of Languages (GRDDL)]), that is to say that a GRDDL-compliant processor can extract useful RDF (as defined in [Resource Description Framework (RDF): Concepts and Abstract Syntax]) representations of the information contained herein.

2 XML Schemas

There are three syntaxes in which [XQuery and XPath Full-Text 1.0] queries can be written. Two are so-called human-readable syntaxes, one in XQuery and one in XPath, defined by [XQuery and XPath Full-Text 1.0] as an extension to the syntax defined in [XQuery 1.0] and the syntax defined in [XPath 2.0], respectively, while the other is an XML syntax for XQuery defined by [XQuery and XPath Full-Text 1.0] as an extension to the XML vocabulary defined in [XQueryX 1.0].

XQueryX, as defined in [XQueryX 1.0] is specified in the namespace http://www.w3.org/2005/XQueryX, which is defined by an XML Schema.

This schema specifies the XML syntax of the XQuery expressions defined in [XQuery 1.0]. It is located at http://www.w3.org/2005/XQueryX/xqueryx.xsd.

The XML syntax defined in [XQuery and XPath Full-Text 1.0] is specified in the namespace http://www.w3.org/2007/xpath-full-text, which extends the XML vocabulary defined in the namespace http://www.w3.org/2005/XQueryX. It is defined by a pair of XML Schemas.

This schema specifies the XML syntax of the Full Text expressions defined in [XQuery and XPath Full-Text 1.0], other than the XML syntax of match options. It is located at http://www.w3.org/2007/xpath-full-text/xpath-full-text-10-xqueryx.xsd.

This schema specifies the XML syntax of match options that are part of the Full Text expressions defined in [XQuery and XPath Full-Text 1.0]. It is located at http://www.w3.org/2007/xpath-full-text/xpath-full-text-10-xqueryx-ftmatchoption-extensions.xsd.

XPath and XQuery Full Text 1.0 queries written using the XML syntax must be valid according to those two schemas. Validation must be performed as defined by [XML Schema Part 1: Structures Second Edition], using strict validation, full validation attempted, using the <xqx:module> element as the validation root. The result of that validation must not be invalid or notKnown.

3 XSLT Stylesheets

Queries written using the XML syntax of [XQuery 1.0] (as defined in [XQueryX 1.0]) have semantics that are defined by the XQuery "human-readable" syntax that results from transforming those XML syntax queries using the stylesheet located at http://www.w3.org/2005/XQueryX/xqueryx.xsl.

Queries written using the XML syntax of [XQuery and XPath Full-Text 1.0] have semantics that are defined by the XQuery "human-readable" syntax that results from transforming those XML syntax queries using the stylesheet located at http://www.w3.org/2007/xpath-full-text/xpath-full-text-10-xqueryx.xsl. This stylesheet extends the stylesheet defined in [XQueryX 1.0] that is located at http://www.w3.org/2005/XQueryX/xqueryx.xsl.
